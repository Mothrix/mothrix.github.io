Semana 1: Introducción al Desarrollo Web

Una página web es un documento de texto con marcas. Las marcas permiten modificar la presentación del documento, incluir elementos no contenidos en el texto (por ejemplo, imágenes), crear hiperenlaces, añadir significado al texto (todavía de forma limitada), etc. A las marcas se les llaman también etiquetas. Las páginas web están pensadas para ser vistas mediante los programas llamados navegadores web.

Semana 2: Introducción al HTML

HTML(Lenguaje de marcado para Hipertextos)

Elemento de construcion basico de una pagina web. HTML usa markup para anotar textos,imagenes y otros contendidos que se muestran en el navegador. Los navegadores son capaces de interpretar las marcas y presentar el documento tal y como lo diseñó el autor.

Semana 3: Introducción al CSS

En inglés CSS significa Cascading Style Sheets, es decir, Hojas de Estilo en Cascada, es un lenguaje de diseño gráfico para definir y crear la presentación de un documento estructurado escrito en un lenguaje de marcado

En las hojas de estilo CSS, los selectores son la parte de las reglas que indican al navegador a qué elementos se van a aplicar las propiedades

  • Selector de tipo: E
  • Selector universal: *
  • Selector de clase: E.e
  • Selector de identificador (id): E#e
  • Selector de descendientes: E F
  • Selector de hijos: E > F
  • Selector de consecutivos: E + F
  • Selector de grupo: E, F

Semana 4: Introducción al JavaScript (DOM, HTTP Y Fetch)

Introducción al JavaScript

Cómo incluir JavaScript: La etiqueta <script> contiene código JavaScript que es automáticamente ejecutado por el navegador una vez que procesa la etiqueta.

Script Externo: Los scripts pueden estar en archivos separados. Se vinculan al HTML con el atributo src en la etiqueta <script>.

Ejemplos:
Ruta absoluta: <script src="/ruta/al/script.js"> </script>
Ruta relativa: <script src="script.js"> </script>
Archivo en la web: <script src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/3.2.0/lodash.js"> </script>

Introducción a DOM (Document Object Model)

La creación del Document Object Model o DOM es una de las innovaciones que más ha influido en el desarrollo de las páginas web dinámicas y de las aplicaciones web más complejas.

DOM permite a los programadores web acceder y manipular los elementos de las páginas HTML a través de objetos llamados nodos.

DOM transforma todos los documentos HTML en un conjunto de elementos llamados nodos, que están interconectados y que representan los contenidos de las páginas web y las relaciones entre ellos.

Los navegadores web transforman automáticamente todas las páginas web en una estructura más eficiente de manipular, el DOM.

HTTP (Protocolo de transferencia de hipertexto)

HTTP (opens in a new tab)(HyperText Transfer Protocol) está diseñado para permitir una comunicación entre clientes y servidores.

HTTP funciona como un protocolo de pedido-respuesta entre el cliente y servidor. Un navegador web puede ser el cliente, y la aplicación en una computadora que aloja un sitio web puede ser el servidor.

Métodos HTTP

  • Método GET: GET es el método HTTP por defecto. Su función es solicitar un recurso específico, por lo tanto las peticiones que utilizan este método solo deben utilizarse para obtener información. Notar que los parámetros de consulta que envía, se encuentran en la URL misma, en forma de pares clave-valor El método GET tiene algunos problemas de seguridad porque los datos insertados son visibles en la URL. Además, solo una cantidad restringida de datos puede ser pasada a través del método GET, ya que la longitud de la URL que un navegador puede atravesar es limitada.
  • Método POST: El método POST se utiliza para enviar una entidad a un recurso en específico al servidor, causando a menudo un cambio en el estado o efectos secundarios del mismo. Generalmente es utilizado para enviar al servidor información que queremos que el mismo "registre".

Fetch (comunicación con el servidor)

JavaScript puede realizar solicitudes a servidores para obtener o enviar información sin necesidad de recargar la página web.

Fetch API: Introducción a la función `fetch()`, que se utiliza para realizar peticiones HTTP desde JavaScript. Permite especificar la URL y opciones como método HTTP, encabezados, etc.

Respuestas de Fetch: El objeto `Response` devuelto por `fetch()` proporciona métodos para acceder al contenido de la respuesta, como `response.text()` para obtener el texto de la respuesta y `response.json()` para interpretar la respuesta como JSON.
